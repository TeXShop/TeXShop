Hay otra manera de componer (‘typeset’) con TeXShop; con este método alternativo, las ilustraciones eps se pueden incluir directamente sin convertirlas previamente a pdf. Para ello, elegir «TeX y DVI» en el menu Componer. Entonces, TeXShop llamará a tex o latex (en lugar de a pdftex o pdflatex) y generará un archivo «.dvi» (en lugar de un «.pdf»), luego llamará a dvips para convertirlo en un archivo postscript, y finalmente llamará a ps2pdf para convertir el archivo postscript a pdf. El método elegido en el menú Componer sólo afectará al archivo tex que se encuentre en ese momento al frente; para otros documentos, TeXShop continuará usando pdftex o pdflatex. Si deseamos cambiar el método predeterminado para componer, podemos hacerlo mediante del diálogo de preferencias.

El método «TeX y DVI» puede ser preferible para proyectos antiguos con muchas ilustraciones en formato eps, para archivos tex con comandos postscript especiales, y para archivos tex que incluyan letras bitmap que no se verán correctamente cuando se compongan mediante pdflatex. La experimentación determinará qué «motor de composición» nos conviene usar.

Hay una manera para establecer de forma permanente en un archivo el método de composición, independientemene de las preferencias establecidas. Si una de las primeras veinte líneas del documento es %!TEX TS-program = tex, entonces se usará el método tex + dvi. Si una de las primeras veinte líneas es %!TEX TS-program = latex, entonces se usará latex + dvi. Si una de las líneas es %!TEX TS-program = pdftex, o %!TEX TS-program = pdflatex, entonces se usará pdftex o pdflatex. Finalmente, si una de las líneas es %!TEX TS-program = personaltex o %!TEX TS-program = personallatex, entonces se usará el correspondiente script personal señalado en el diálogo de preferencias.
