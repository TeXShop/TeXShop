La sincronización es una importante utilidad incorporada en Textures, una implementación comercial de TeX para Macintosh Classic desarrollada por Blue Sky TeX Systems. Al usar la sincronización, un autor puede hacer clic en la ventana del dvi previsto e, inmediatamente, el cursor aparece en el correspondiente lugar de la ventana de código. Blue Sky ha prometido una versión nativa para MacOS X de Textures, pero todavía no la ha lanzado.La implementación de la sincronización en Textures es admirada por su habilidad para localizar el correspondiente punto en el código.

Desde esa implementación, varios sistemas TeX han incorporado, aproximadamente, la misma utilidad. Para ello han usado estilos adicionales que también hacían que TeX escribiera información extra en el archivo dvi. En verano de 2003, Piero D'Ancona escribió un archivo de estilo similar en cierto sentido, pero para pdflatex. Este paquete, pdfsync.sty, hace que pdftex escriba, al componer páginas, un archivo extra, en el cual guarda la información necesaria para lograr la correlación entre los archivos de entrada (código tex) y salida (pdf). El trabajo de D'Ancona depende de comandos internos de pdftex que sacan las coordenadas x e y de ciertas operaciones de composición.

TeXShop 1.35 usa la última versión de pdfsync.sty. Lee «¿Cómo configuro TeXShop?: ¿Has instalado TeXShop correctamente?» para estar seguro de si lo has hecho bien. La sincronización que proporciona de pdfsync sólo están disponibles con pdftex, pdflatex y context; no funcionan si usas tex + ghostscript para componer.

Para usar pdfsync con LaTeX, añade al preámbulo de tu documento con código tex (antes de la línea con el \begin{document}) la siguiente línea:

\usepackage{pdfsync}
Este paquete será ignorado si estás compilando con tex + ghostscript, pero si lo estas haciendo con pdflatex, se creará un archivo extra con extansión .pdfsync. Por ejemplo, si el documento con el códido fuente se llama main.tex, el archivo extra se llamará main.pdfsync. Este archivo contiene la información necesaria para relacionar puntos del archivo pdf con puntos del archivo tex. Cuando se usa pdfsync, la mayoría de los archivos se compilan sin errores, pero, en un pequeño número de casos, este paquete introduce errores extra. Si eso ocurre, el código peligroso se puede encerrar entre un par de comandos \pdfsyncstop y \pdfsyncstart. Por otra parte, para aumentar la precisión, se pueden añadir puntos de syncronización extra mediante el comando \pdfsync.

Una vez que has compuesto las páginas a partir de un archivo con pdfsync, haz clic en un punto del pdf previsto mientras mantienes pulsada la tecla comando. Aparecer‡ en el frente la ventana con el código, allí estará seleccionada la línea que contiene el correspondiente código, y la ventana se desplazará hasta mostrar esa línea si fuera necesario. Si un proyecto contiene un archivo raíz y capítulos adicionales añadidos con \include, y si el punto en la ventana del pdf corresponde al código incluido en el archivo de algún capítulo, el archivo se abrirá, aparecerá en el frente, y se producirá el desplazamiento hasta el lugar apropiado en el código.

Esta sincronía también funciona en sentido contrario. Haz click en una línea del código fuente mientras mantienes pulsada la tecla comando. Entonces, TeXShop muestra la ventana con el pdf previsto, y selecciona la página apropiada. Este proceso funciona si el documento con el código tiene una línea %SourceDoc o un archivo raíz; se selecciona la página apropiada en la ventana con el pdf del documento raíz. Aviso: si un documento raíz usa \include para añadir otros archivos de código, pero estos archivos no saben cuál es su documento raíz (es decir, no tienen línea %SourceDoc o archivo root), entonces comando-click nos desplaza desde la ventana con el pdf a la del código incluido, pero no puede volver de la del código incluido a la del pdf. Esto se remedia fácilmente añadiendo el correspondiente comentario %SourceDoc al comienzo de cada archivo con código.

En esta implementación inicial, la línea de código que se muestra será aproximadamente la correcta, pero la real puede encontrarse una pocas líneas más arriba o más abajo. Con el tiempo, el código de sincronización de TeXShop será pedurado, y la precisión mejorará.

Tal como se ha explicado antes, pdfsync funciona con archivos que se cargan con \include{myfile}. También funciona cuando el modo que se usa para incluir archivos es \input{myfile}; pero la sintaxis alternativa \input myfile no funciona.

El archivo pdfsync.sty se usa en LaTeX; los archivos similares pdfsync.tex y pdfsync4context.tex sirven para la sincronización con tex y context.

La sincronización funciona por medio de un archivo mysource.pdfsync en el que se almacenan los datos correspodientes a ciertos puntos de sincronización en la ventana con el pdf previsto. Grosso modo, cada entrada de datos contiene

el número de la página del documento pdf en el que se ha colocado el punto
la localización del punto en esa página
el nombre del archivo de código que ha producido el punto
el número de línea en ese documento de código que ha producido el punto

Hay una manera de indicarle a TeXShop que muestre esos puntos de sincronización. La barra de herramientas de la ventana del pdf contiene un ‘checkbox’ llamado SyncMarks. Por defecto, este ítem no aparece; usa «Personalizar barra de herramientas...» en el menú ventana para ponerlo. Cuando el ‘checkbox’ está marcado, TeXShop muestra los puntos de sincronización.

Cuando la ventana con el pdf previsto aparece por primera vez, este ítem no está marcado. Esto se puede cambiar con una preferencia oculta:

defaults write TeXShop ShowSyncMarks YES
