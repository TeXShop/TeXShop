ConTeXt y MetaPost

ConTeXt es un paquete de macros de propósito general para TeX, desarrollado por Hans Hagen; para algunos es una alternativa seria a LaTeX. Ver detalles en http://www.pragma-ade.com/. Se puede encontrar una útil introducción en http://berend.gameren.nl/tex/, en el documento «LaTeX in proper ConTeXt». TeXShop soporta ConTeXt.

MetaPost es un sistema similar a MetaFont, desarrollado por John Hobby, y que obtiene como salida archivos postscript y pdf. Se puede usar para producir detalladas ilustraciones en PostScript. Ver detalles en http://cm.bell-labs.com/who/hobby/MetaPost.html. Además, se pueden encontrar interesantes ejemplos elaborados con MetaPost en muchos sitios: ver http://www.cs.ucc.ie/~dongen/mpost/mpost.html. TeXShop soporta MetaPost.

He aquí un ejemplo de un archivo MetaPost:

prologues:=2;
color yellow; yellow = green + red;
def star (expr size, n, pos, color)=
for a=0 step 360/n until 360:
draw (origin -- (size/2,0))
rotatedaround (origin, a)
shifted pos withcolor color;
endfor ;
enddef;

beginfig(-1);
pickup pencircle scaled 2mm; star (2cm,5,origin,red);
endfig;

beginfig(2);
pickup pencircle scaled 2mm; star (2cm,7,origin,yellow);
endfig;

beginfig(3);
pickup pencircle scaled 2mm; star (2cm,11,origin,green);
endfig;

beginfig(4);
pickup pencircle scaled 2mm; star (2cm,13,origin,blue);
endfig;

end

Supón que este archivo se llama «metademo.tex». Al procesarlo con MetaPost, generará cuatro archivos postscript diferentes, llamados metademo.ps, metademo.2, metademo.3, metademo.4. Estos nombres están determinados por el parámetro numérico de «beginfig()». Si este número es positivo, como beginfig(10), el archivo resultante se llamará metademo.10. Si este número es negativo, como beginfig(-10), el archivo resultante se llamará metademo.ps, sobreescribiendo cualquier archivo metademo.ps creado previamente.

TeXShop asume que uno de los números en beginfig() es negativo, y así que MetaPost generará un archivo llamado metademo.ps. Entonces, TeXShop llama a epstopdf para convertir este archivo postscript en un archivo pdf, y muestra el archivo pdf resultante. Si ningún parámetro de beginfig es negativo, MetaPost se ejecutará hasta el final sin error, pero TeXShop mostrará un error cuando intente convertir metademo.ps en un archivo pdf.

Cuando estés editando un archivo MetaPost, cambia el número de la figura que estés editando de positivo a negativo. TeXShop mostrará esta figura como si estuviera siendo depurada (‘debugged’). Cuando estés satisfecho con la figura, vuelve a poner el número positivo, y cambia el número de otra figura de positivo a negativo.

TeXShop puede convertir archivos postscript como metademo.2 a pdf, pero primero hay que renombrar estos archivos como metademo.2.ps, para que TeXShop reconozca que son archivos postscript.

Una vez que los archivos MetaPost están creados, pueden usarse y visualizarse como cualquier otra ilustración. Se puede usar Pdflatex si las ilustraciones están ya en formato pdf, o TeX y Ghostscript para incluir ilustraciones postscript sin convertir. Por ejemplo, podremos ver las cuatro ilustraciones creadas con el código MetaPost anterior compilando el siguiente archivo mediante TeX y Ghostscript:

\documentclass[11pt] {article}
\usepackage{graphicx}

\begin{document}

Here are some illustrations.
\vspace{.2in}

\includegraphics[width=1cm]{metademo.1}
\hfill
\includegraphics[width=1cm]{metademo.2}
\hfill
\includegraphics[width=1cm]{metademo.3}
\hfill
\includegraphics[width=1cm]{metademo.4}
\hfill

\end{document}
