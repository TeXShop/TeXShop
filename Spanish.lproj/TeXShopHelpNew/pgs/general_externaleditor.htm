Uso de un editor externo

Podemos configurar TeXShop para ser usado con un editor externo. Cuando éste es el caso, aparece un nuevo menú llamado «Abrir para previsualizar...». Al usarlo, el programa nos indicará que elijamos un archivo con código fuente tex. Pero sólo se abrirá el archivo pdf asociado, en su correpondiente ventana (si no hemos compilado aún el código, la ventana con el pdf estará en blanco). Gracias a la barra de herramientas de Leuski, la ventana con el pdf previsto se puede configurar de manera que contenga un botón para componer. Puedes abrir el código fuente con cualquier editor. Cuando quieras componer lo escrito, guarda los cambios en tu editor, vuelve a la ventana del pdf previsto y compila. De este modo, TeXShop nunca abrirá ni modificará el archivo con el código. Simplemente hará de intermediario para enviarlo al correspondiente proceso TeX o LaTeX en Unix.

Para los usuarios que prefieran usar, generalmente, un editor externo, existe en las preferencias un ítem llamado «Configurar para editor externo». Si hemos elegido esta opción, los menús «Abrir...» y «Documentos recientes» abren los archivos de la manera anteriormente descrita, para editarlos con un editor externo. Además, «Abrir para previsualizar...» se sustituye por «Abrir para editar...», y sirve para abrir el código con el editor interno de TeXShop, para las raras ocasiones en que queramos usar el editor interno. Los menús «Abrir...» y «Documentos recientes» también pueden abrir archivos jpg, tiff, ps, pdf, dvi y log, entre otros; tener seleccionada la preferencia «Abrir para previsualizar...» no cambia el comportamiento de TeXShop con estos archivos.

El soporte AppleScript para la opción de editor externo es todavía débil; será mejorado en alguna versión futura, haciendo posible componer completamente el documento desde un editor que maneje AppleScript.
